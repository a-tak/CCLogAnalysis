# Claude Code プロジェクトガイドライン

## パブリック公開プロジェクトの注意事項

このプロジェクトはGitHubでパブリック公開される予定です。以下のルールを遵守してください。

### 個人情報の取り扱い

ドキュメントやコード例で以下の情報を含めないでください：

#### ❌ 禁止事項

- **ユーザー名**: 実際のユーザー名
- **プロジェクト固有の名前**: 実際のプロジェクト名
- **実際のパス**: 個人のディレクトリパス
- **個人を特定できる情報**: メールアドレス、実名など

#### ✅ 代わりに使用するもの

- **ユーザー名** → `{username}` または `user`
- **プロジェクト名** → `my-project` または `{project-name}`
- **パス** → 一般化されたパス例（例: `C:\Users\{username}\...`）
- **ブランチ名** → `feature-branch`, `main`, `develop` などの一般的な名前

### 例

#### ❌ 悪い例

```bash
cd C:\Users\(実際のユーザー名)\.claude\projects\(実際のプロジェクト名)
```

#### ✅ 良い例

```bash
cd C:\Users\{username}\.claude\projects\{project-name}
```

## プロジェクト情報

### 作業進捗の確認

- **WIPドキュメント**: `docs/WIP/` フォルダに日付ごとの作業進捗を記録
  - 最新のWIPドキュメントを参照して、現在の開発状況を把握してください
  - 新しいセッションで作業を開始する際は、必ずWIPドキュメントを確認
  - 作業完了後は、新しいWIPドキュメントを作成して進捗を記録

### ドキュメント構成

- `README.md`: プロジェクト概要・セットアップ手順
- `docs/要件.md`: 要件定義
- `docs/ログフォーマット調査.md`: Claude Codeログ形式の調査結果
- `docs/技術スタック.md`: 技術選定の詳細
- `docs/API設計.md`: REST API仕様書
- `docs/WIP/`: 作業進捗ドキュメント（Work In Progress）
- `.claude/rules/`: 開発ルール（モジュール別）

## 開発ガイドライン

### テスト駆動開発（TDD）

- **必須**: すべての新機能は「テスト→実装→リファクタリング」のサイクルで開発
- 詳細は `.claude/rules/general.md` を参照

### プランファイル

- 詳細は `.claude/rules/plans.md` を参照

### ドキュメント作成時

- サンプルコードやログ例を記載する際は、必ず個人情報を一般化する
- 実際の調査データを使用する場合は、公開前に必ずレビューする
- ドキュメントは常に最新状態を維持

### サーバー起動・停止

- **必須**: サーバーの起動・停止は `/server-management` スキルを使用する
- コマンドラインで直接起動・停止しない（PID管理やポート競合を避けるため）
- 起動: `/server-management start dev` または `/server-management start prod`
- 停止: `/server-management stop`
- 詳細は `README.md` の「実行」セクションを参照

### コミット前チェックリスト

- [ ] ユーザー名が含まれていないか
- [ ] プロジェクト固有の名前が含まれていないか
- [ ] 実際のパスが含まれていないか
- [ ] その他の個人情報が含まれていないか
- [ ] テストが全てパスしているか
