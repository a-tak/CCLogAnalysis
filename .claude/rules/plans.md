# プランファイル管理ルール

## 📍 重要: このルールは必ず守ってください

`.claude/plans/` フォルダ内のプランファイルを作成・参照する際は、このルールを厳守してください。

---

## プランファイル命名規則

### 命名形式

```
YYYY-MM-DD_タスク名.md
```

### ルール詳細

- **必須**: プランファイルは `.claude/plans/` ディレクトリに配置
- **日付**: ISO 8601形式（YYYY-MM-DD）を使用
- **タスク名**: 日本語で記述し、わかりやすく具体的に
- **区切り文字**: アンダースコア（`_`）を使用

### ✅ 正しい例

```
.claude/plans/2026-01-24_ファイル監視機能実装.md
.claude/plans/2026-01-25_API認証機能追加.md
.claude/plans/2026-01-26_データベーススキーマ修正.md
```

### ❌ 間違った例

```
.claude/plans/gleaming-swimming-locket.md  # ランダムな名前
.claude/plans/plan.md                       # 日付がない
.claude/plans/2026-01-24-task.md           # 日本語でない、区切り文字が違う
.claude/plans/add-feature.md                # 日付がない、日本語でない
```

---

## プランファイル作成時の注意事項

### 1. 新しいプランを作成する場合

- EnterPlanMode を使用してプラン作成を開始
- プランファイル名は必ず命名規則に従う
- 作成日時を正確に反映

### 2. 既存のプランファイルがある場合

- **まず確認**: `.claude/plans/` 内のファイル一覧を確認
- **命名規則違反を発見した場合**:
  - ユーザーに報告
  - 適切な名前への変更を提案

### 3. プランの内容

- タスクの目的と背景を明確に記述
- 実装手順を具体的にリストアップ
- 影響範囲とリスクを明記
- テスト計画を含める

---

## チェックリスト

プランファイルを作成・修正する際は、以下を確認してください：

- [ ] ファイル名が `YYYY-MM-DD_タスク名.md` の形式になっている
- [ ] 日付が正しい（作成日時を反映）
- [ ] タスク名が日本語で具体的に記述されている
- [ ] `.claude/plans/` ディレクトリに配置されている
- [ ] プラン内容が具体的で実行可能である

---

## 既存ファイルの修正

命名規則に違反しているプランファイルを発見した場合：

1. ユーザーに報告
2. 適切な名前を提案
3. ユーザーの承認後、ファイル名を変更
4. 関連するドキュメントへの参照も更新
